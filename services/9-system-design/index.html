System Design

The system device can be subdivided into three categories and their integration. The three categories are: mechanical components, electrical components, and software components. Each subsystem can be broken down into its respective subcomponents, the process is iterative until base metrics are purchasable or finitely small actionable items.

Mechanical subsystems:

The mechanical subsystems include all physical components for the device as aswell as all interfaces to the device. The crossover for the mechanical subsystem also includes the hardware component integration that is designed from the electrical subsystem. Therefore the mechanical subsystem includes: plastics manufacturing for the shell, thermal cooling void space, internal mounting for the battery, internal mounting for the PCB, wiring diagram, PCB interfaces with the shell via buttons and LEDs, PCB interfaces with external power charging, external mounts to the plastic shell, strap design, strap mount, and proper design of the low force strap breakaway clip.

Mechanical implementation of subsystems:

The majority of the mechanical designs revolve around the plastic mold injected shell. Allowing the shell to be plastic model injected allows for many of the mechanical requirements to be manufactured into the product. This includes: the external shell, external strap mounts, through holes for LEDs, buttons, power input holes, battery mount, PCB mount and thermal cooling void space. A majority of the design should be ensuring that the plastic shell can meet all of these various requirements. As well as meeting all required engineering standards. This should all be done on a professional CAD such as Solidworks. The straps and clip can be designed on the same CAD program. 3D printing prototypes is a must, as it can save time and money on misdesign before mass production.

Electrical subsystems:

The electrical subsystems include: PCB design and manufacturing, ensuring power requirements and power consumption levels are appropriate, ensuring hardware is adequate for meeting software designs, building in hardware margin for software change field updates.

Once all of the software components have dictated required hardware components, a PCB board should be designed and test manufactured by popular single board manufacturers such as JLCPCB. One all of the requirements are set forth, hardware edge testing is mandatory and revealing on product useability. A four layer board should be the standard for simplicity and cost, but other cost parameters should be taken into account for re-engineering hardware if it is economical.

Software subsystems:

The software subsystems include: data organization onboard PCB, data buffering, data transfer, implementing software to handle all trigger events, such as button pushes, low battery, no sensor readings, and creating an expandable software base that allows for alteration and updates.

Software implementation of subsystems:

All of the software should have a focus on speed, reliability, and useability. While programming languages like python should be used in the early design/proof of concept. The final product should be running C/C++ code. And should be designed to run in a functional way when necessary and an object oriented way when ambiguous. The software should also accommodate future remote upgrades.
